## Overview

All assets are preloaded in the `<a-assets>` container to ensure smooth performance and prevent stuttering during gameplay. Assets are referenced throughout the scene using ID selectors (e.g., `#img1`, `#carpet`).

```html index.html:9-19
<a-assets>
  <img id="img1" src="img/img1.jpg">
  <img id="img2" src="img/img2.jpg">
  <img id="img3" src="img/img3.jpg">
  <img id="img4" src="img/img4.jpg">
  <img id="back" src="img/atras.jpeg">
  <img id="carpet" src="piso.jpg">
  <img id="stone" src="lados.jpg">
  <audio id="click-sound" src="cartas.mp3"></audio>
  <audio id="background-music" src="fondo.mp3" preload="auto"></audio>
</a-assets>
```

## Asset Loading

### Preloading Behavior

A-Frame's `<a-assets>` system ensures all assets are loaded before the scene renders:

1. **Automatic blocking**: Scene waits for all assets to load
2. **Timeout**: Default 3-second timeout (can be customized)
3. **Progress events**: Can be monitored via `loaded` event

<Info>
  The `preload="auto"` attribute on background music ensures it's fully loaded before gameplay begins, preventing audio stuttering.
</Info>

## Asset Modifications

### Dynamic Asset Swapping

Card textures are swapped dynamically during gameplay:

```javascript
// Flip to card face
setTimeout(() => card.setAttribute('src', '#' + card.dataset.id), 125);

// Flip back to card back
card.setAttribute('src', '#back');
```

<Note>
  The 125ms delay when revealing cards creates a smooth half-flip transition before the texture changes.
</Note>

## Asset Naming Convention

### Spanish Names

Many assets use Spanish filenames, reflecting the game's linguistic theming:

- `atras.jpeg` = "back" (card back)
- `cartas.mp3` = "cards" (card sound)
- `fondo.mp3` = "background" (background music)
- `piso.jpg` = "floor" (floor texture)
- `lados.jpg` = "sides" (wall texture)

<Info>
  In-game text is also in Spanish ("VIDA", "PERDISTE TU ALMA", "SALISTE CON VIDA"), creating a cohesive linguistic experience.
</Info>
