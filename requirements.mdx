## Overview

Forsaken Casino is built on A-Frame 1.4.0, a web framework for building virtual reality experiences. This page outlines the technical requirements for running the game.

<Info>
The game runs in both standard web browsers (desktop mode) and VR headsets (immersive mode).
</Info>

## A-Frame Framework

The game is built with A-Frame 1.4.0, loaded via CDN:

```html index.html
<script src="https://aframe.io/releases/1.4.0/aframe.min.js"></script>
```

<Note>
A-Frame 1.4.0 requires no installation or build process. The framework is loaded directly from the CDN.
</Note>

## Technical Dependencies

### Core Framework

The game uses A-Frame's built-in components:

```html index.html
<a-entity id="camera" camera look-controls wasd-controls="acceleration: 10" rotation="-25 0 0">
  <a-cursor id="cursor" raycaster="objects: .clickable" material="color: #ff0000" scale="0.5 0.5 0.5"></a-cursor>
</a-entity>
```

**Key A-Frame Components:**
- `camera`: Camera rig for viewing the scene
- `look-controls`: Mouse/headset look controls
- `wasd-controls`: Keyboard movement (desktop)
- `raycaster`: Interaction detection for clickable objects
- `sound`: 3D positional audio support

### Asset Loading Requirements

The game preloads assets using A-Frame's asset management system:

```html index.html
<a-assets>
  <img id="img1" src="img/img1.jpg">
  <img id="img2" src="img/img2.jpg">
  <img id="img3" src="img/img3.jpg">
  <img id="img4" src="img/img4.jpg">
  <img id="back" src="img/atras.jpeg">
  <img id="carpet" src="piso.jpg">
  <img id="stone" src="lados.jpg">
  <audio id="click-sound" src="cartas.mp3"></audio>
  <audio id="background-music" src="fondo.mp3" preload="auto"></audio>
</a-assets>
```

<Warning>
**CORS Requirement:** Assets must be served from an HTTP server, not the file:// protocol. Cross-origin assets require proper CORS headers.
</Warning>

## Performance Considerations

### Scene Complexity

The game maintains optimal performance through efficient scene design:

```html index.html
<a-scene fog="type: exponential; color: #0a0000; density: 0.15">
  <a-light type="ambient" color="#444"></a-light>
  <a-light id="flicker" type="point" intensity="1" position="0 3 0" color="#f00"
           animation="property: intensity; from: 0.2; to: 1.2; dur: 150; loop: true; dir: alternate"></a-light>
</a-scene>
```

**Performance Features:**
- Minimal polygon count (8 cards + environment)
- Optimized lighting (1 ambient + 1 animated point light)
- Exponential fog to reduce draw distance
- Efficient raycasting (only targets `.clickable` elements)

## WebXR API Support

### Checking WebXR Availability

To verify WebXR support in your browser, run this in the console:

```javascript
if (navigator.xr) {
  navigator.xr.isSessionSupported('immersive-vr').then((supported) => {
    console.log('WebXR VR supported:', supported);
  });
} else {
  console.log('WebXR not available');
}
```

### Enabling WebXR in Browsers

<CodeGroup>
```text Chrome
1. Navigate to chrome://flags
2. Search for "WebXR"
3. Enable "WebXR Device API"
4. Restart browser
```

```text Firefox
1. Navigate to about:config
2. Search for "dom.vr.webxr.enabled"
3. Set to true
4. Restart browser
```
</CodeGroup>

## Network Requirements

### CDN Dependencies

The game requires internet access to load A-Frame:

```html index.html
<script src="https://aframe.io/releases/1.4.0/aframe.min.js"></script>
```

**Size:** ~400 KB (minified)

<Info>
For offline deployment, download `aframe.min.js` and serve it locally.
</Info>

### Local Server Setup

The game must be served via HTTP/HTTPS:

<CodeGroup>
```bash Python
python -m http.server 8000
```

```bash Node.js
npx serve -p 8000
```

```bash PHP
php -S localhost:8000
```
</CodeGroup>

### Network Access for VR Headsets

To access the game from a standalone VR headset:

1. Connect your computer and VR headset to the same WiFi network
2. Find your computer's local IP address:
   ```bash
   # Windows
   ipconfig
   
   # macOS/Linux
   ifconfig
   ```
3. Start the local server on your computer
4. In your VR headset's browser, navigate to:
   ```
   http://<your-ip-address>:8000
   ```

<Warning>
Ensure your firewall allows incoming connections on port 8000.
</Warning>

## Troubleshooting

### WebGL Not Available

If you see a WebGL error:

1. Update your graphics drivers
2. Enable hardware acceleration in browser settings
3. Try a different browser (Chrome/Firefox recommended)

### VR Button Not Appearing

The VR button appears automatically when:
- WebXR is supported
- A VR headset is detected
- The page is served over HTTPS (or localhost)

### Performance Issues

If experiencing low frame rates:

1. Close other browser tabs
2. Update graphics drivers
3. Lower browser window resolution
4. Disable browser extensions

### CORS Errors

If assets fail to load:

```
Access to image at 'file:///...' from origin 'null' has been blocked by CORS policy
```

**Solution:** Always use an HTTP server, never open `index.html` directly.

## Browser Feature Detection

The game relies on these browser APIs:

- ✅ **WebGL:** Required for 3D rendering
- ✅ **Web Audio API:** Required for sound effects
- ✅ **requestAnimationFrame:** Required for animations
- ⚠️ **WebXR Device API:** Required for VR mode only
